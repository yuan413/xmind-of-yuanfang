# Plan: easycode-java-orm 浏览器表单支持项目切换与级联候选加载

## Context
当前 easycode-java-orm 浏览器表单只针对当前 projectRoot 预加载 bizPackageName / dataSource 候选，并且没有 projectName 选择与级联更新机制（form_capture_server.py 中 build_shell_html/build_easycode_fields/do_GET）。新需求希望：项目名称可选，默认当前项目；候选项目来自当前项目同级目录扫描；当用户更换项目名称时，bizPackageName/dataSource 候选在切换后再加载，避免一次性预加载所有项目。

## Goals
- 浏览器表单新增 projectName 选择，默认值为当前 projectRoot.name。
- projectName 候选来源：扫描当前项目所在目录的同级项目。
- 当 projectName 变化时，动态加载该项目的 bizPackageName/dataSource 候选并更新 UI（不预加载所有项目）。
- 选中的 projectName 影响后续生成文件的落盘目标（切换后生成到选中项目目录）。
- AskUserQuestion 模式也增加 projectName 选择并可切换项目候选。
- bizPackageName/dataSource 改为单一输入框（下拉候选 + 手动输入，使用 datalist 方案）。
- 回归测试使用 sds-schedule-dispatch-center，覆盖切换项目与提交 payload。

## Plan
1. 发现与校验项目列表
   - 在 discover_main_interaction_context.py 增加 `list_sibling_projects(project_root)` 与 `resolve_project_root(project_root, project_name)`。
   - 规则：严格过滤，仅目录且至少包含 `src/main/java` 或 `config/productiondocker/application-druid.properties`。
   - 输出 projectCandidates（value/label/preview），供浏览器表单使用。

2. 后端按需候选查询
   - 在 form_capture_server.py 增加 `/candidates?projectName=...` GET endpoint。
   - 对 projectName 做白名单校验（来自 projectCandidates）。
   - 调用 `build_bootstrap_payload(selected_project_root)` 仅计算该项目的 `serviceCandidates` 和 `datasources`，返回 JSON。

3. 表单 UI 增加 projectName 字段
   - HTML_TEMPLATE 新增 projectName 选择区（置于表单顶部）。
   - build_shell_html 增加 projectName 的默认值与选项渲染（或通过 candidateData 传递并在前端渲染）。
   - build_easycode_fields 增加 projectName 字段，确保 /submit 收集该值。
   - 前端 JS：在 projectName change 时 fetch `/candidates`，更新 bizPackageName/dataSource 列表与预览，展示加载/错误状态，并在必要时清空旧选择。
   - bizPackageName/dataSource UI 改为单一输入框（input + datalist），支持候选下拉与自由输入。

4. 默认与兼容
   - 页面首次加载仅包含当前项目候选（保持现有行为）。
   - projectName 未变更时行为与现有一致。
   - 若所选项目缺少候选，则保持可手动输入。
   - form_submit 需要把 projectName 回传，后续生成逻辑应基于该项目 root 继续（包含 derive 与 generate）。

5. Bash fallback（无 Python）
   - 在 fallback_scan_candidates.sh 增加可选 `--project-name` 参数，用于扫描指定项目。
   - 增加 projectCandidates 输出，供无 Python 时的 AskUserQuestion 交互（若决定支持 projectName）。

6. 文档同步
   - SKILL.md：补充 projectName 选择与级联加载说明，明确不预加载所有项目，并说明切换后生成到所选项目；更新 bizPackageName/dataSource 输入方式为单一输入框 + 候选下拉。
   - interaction-patterns.md：更新浏览器表单要点，说明 projectName 位置与切换后加载行为，以及单一输入框交互。
   - AskUserQuestion 字段骨架：加入 projectName 字段及其候选来源与说明；bizPackageName/dataSource 提示为可手动输入。

## Verification
- 启动浏览器表单：projectName 默认当前项目，候选加载与现有一致。
- 切换 projectName：前端触发 `/candidates` 请求，仅此时加载对应项目候选；UI 正确更新并显示加载/失败提示。
- bizPackageName/dataSource 为单一输入框 + 候选下拉（datalist），支持自由输入与候选选择。
- /submit 输出包含 projectName；bizPackageName/dataSource 对应选中项目；生成文件落盘路径位于该项目目录。
- 无 Python 场景（若支持 projectName）：fallback 输出 projectCandidates 且可按项目名扫描候选。
- 确认未预加载所有项目的候选数据（仅当前项目在首屏载入）。
- 回归测试：在 sds-schedule-dispatch-center 使用 Playwright 覆盖切换项目与提交 payload。
